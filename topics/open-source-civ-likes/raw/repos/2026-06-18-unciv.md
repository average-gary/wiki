# Unciv — yairm210/Unciv

Open-source, moddability-focused Android and Desktop **remake of Civ V**.
Kotlin + libGDX. Highest-velocity OSS civ-like by a wide margin.

## Stack

- **Kotlin 99.5%**, libGDX framework
- License **Mozilla Public License 2.0 (MPL-2.0)** — uncommon in this space;
  weaker copyleft than GPL, friendly to mod authors
- Modular layout: `android/`, `desktop/`, `core/`, `server/` subprojects
- **JSON-only** ruleset format for mods

## Platforms

Android (Play, F-Droid), Windows MSI, Linux Flatpak/AUR, macOS Brew,
Raspberry Pi, web via Docker/VNC. **iOS explicitly not planned.**

## Activity (as of 2026-06-18)

- **12,999 commits, 940 releases**

- **5 point releases (4.20.10–4.20.14) in ~2.5 weeks** (May 31 – Jun 16,

## Copyright posture (project's own statement)

Unciv openly describes itself as a **"remake of Civ V"** — not "inspired
by," not generic 4X.

Legal defense rests on **US Copyright Office circular FL-108**: *intellectual
property rights do not apply to mechanics.* Game mechanics are not
copyrightable; only expression is.

Project explicitly admits two off-limits items:
1. *"definitely illegal to use any assets from the original game (images,
   sound etc) - they belong to Firaxis"* → all assets must be replaced
2. Using *"the Civilization name"* or impersonating Civ branding → *"probably
   illegal"*

Trademark protection for "Civilization" is treated as **limited to the
logo, not the word** — a position that has not been litigation-tested.

Project deliberately refuses to add **non-Civ-V features** — scope discipline
as a legal-defense posture. Don't make Firaxis feel like you're competing
on new ground.

## Mod ecosystem

- In-game Mod Manager → "Download mod from URL" → paste GitHub repo
  → auto-extracted
- **GitHub-as-mod-registry**: modders tag repos with `unciv-mod` topic; Mod
  Manager queries GitHub for that topic
- File format: **JSON only**. Two folders: `/jsons` (game data), `/Images`
  (graphics). No compilation or programming.
- Two mod classes:
  - **Extension mods** add to existing rulesets — recommended starting point
  - **Base ruleset mods** set `"isBaseRuleset":true` in `ModOptions.json`
    and replace everything

## Why this matters

Unciv's Kotlin/libGDX + JSON-only modding outpaces C-based incumbents in
release cadence. The MPL-2.0 license + GitHub-topic mod registry is the most
accessible 4X modding platform in OSS. The copyright posture (FL-108 +
asset-replacement boundary) is the canonical legal stance for civ-clones
and worth its own article.
