# PR #8219: Add statsvg_rs to Project/Tooling Updates (closed-not-merged)

Editorial precedent confirming the bare-link rule and the link-text-length norm in practice.

## Editor feedback (paraphrased / quoted)
- "the descriptive text for the link is too long. Could you limit it to around 5 words?"
- "We do not accept links solely to GitHub repos or to crates.io pages. Please see our guidelines for more info."

## Outcome
Closed without merge; editor invited a follow-up the next week with a shortened title and substantive description (e.g., a blog post wrapping the project).

## Companion data point
PR #8200 ("Announcing stdx, Rust's extended library") closed for the same reason — title too long; resubmitted as #8250 successfully.

## Implications for Rust-Bitcoin submissions
- Don't link directly to the rust-bitcoin GitHub repo, BDK GitHub repo, or LDK crates.io page.
- Always link to a blog post / release notes page / deep-dive article, never the bare repo or crate.
- Title text should be ~5 words; verbose descriptive sentences are flagged by editors.
