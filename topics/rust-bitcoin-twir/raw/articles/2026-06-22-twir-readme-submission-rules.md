# TWiR README — Editorial / Submission Rules

Canonical rules live in `README.md` on `main` of `rust-lang/this-week-in-rust`. There is no separate CONTRIBUTING.md.

## What we look for
- How-to intros and advanced deep dives into Rust concepts/areas
- Walkthroughs explaining concepts in different ways than the Rust book / Rustlings / Rust by Example
- Updates on tooling **when in long form or framed as a tutorial**
- Rust-related podcasts, screenshots, videos, slides, meetup recordings/announcements
- Observations and thoughts on Rust and the Rust community
- Calls for participation in Rust OSS projects
- Rust job announcements

## What we do NOT include
- Anything violating the [Rust CoC](https://www.rust-lang.org/policies/code-of-conduct)
- Rants / degrading content
- Duplicates of recent posts (even with wording changed slightly)
- Anything behind a paywall (incl. Medium members-only)
- Anything that requires email/info capture to access

## Project/Tooling Updates section — additional rules
- **One submission per contributor per week.**
- Updates must be in long form **or framed as a tutorial** — high amount of Rust-specific info (examples in Rust, lessons learned).
- Release notes/changelogs accepted **only if they include more than just the changelog** — why the changes were made, how to use them, lessons from making them.
- LLM-assisted writing acceptable **only with disclosure**.
- **Bare GitHub or crates.io links are rejected.** "There are too many being created/updated every week for us to include them all."

## LLM policy (verbatim)
> If you submit an LLM-written article to TWiR, we request that the LLM authorship be disclosed in the article. ... If the author is an LLM, the truthfulness of its output may be compromised, and that's a problem. But more importantly, there is no community growth happening: there is no author to learn from community feedback, and the reader is not able to connect with a peer.

## Operational notes
- Drafts live in `draft/YYYY-MM-DD-this-week-in-rust.md`.
- Each link in the format `* [Title of the linked Page](URL)`.
- Video/audio prefix: `[video]` or `[audio]`.
- Editor team listed in README; Language Reviewers handle non-English.
