# epdiy (vroland)

Defines the boundary: **the wrong tool for a Waveshare SPI driver board**, but explains why big/grayscale panels push you to ESP32-S3 + PSRAM.

- Drives **parallel (not SPI)** e-paper panels salvaged from e-readers: ED060, ED050, ED097 (9.7"), ED047TC1 (4.7", LilyGo), ED078KC1 (7.8"), ED133UT2 (13.3").
- Targets ESP32 (V2–V6 boards) and **ESP32-S3 (V7, much faster)**; ESP-IDF + Arduino.
- Provides **4bpp / 16-level grayscale** — which SPI panels + GxEPD2 generally cannot.
- **Incompatible with the Waveshare SPI driver board / GDEW-GDEY SPI panels** — different interface entirely. Large parallel panels effectively require **PSRAM** for framebuffers.

Note: some calendar projects (Stavros Timeframe, Fridge-Calendar) use epdiy — but those pair it with e-reader parallel panels or ESP32-S3, not the Waveshare SPI board.
