# GxEPD2 (ZinggJM)

The dominant Arduino SPI e-paper library. Built on **Adafruit_GFX**; pairs with **ArduinoJson** for data-driven dashboards.

## Core mechanics
- **Paged drawing** to cope with limited RAM: buffer height is a template parameter `page_height`. Set `page_height == display height` for full buffering (needs enough RAM); otherwise the draw callback runs once per horizontal strip (page). This is what lets a **PSRAM-less WROOM-32E drive large panels** — trading RAM for redraw time.
- Three draw patterns: **picture loop** (`firstPage()`/`nextPage()`), **`drawPaged()`** callback, and **full-buffer** direct draw.
- Update modes: full refresh (slow, clean) vs **partial/window update** via `setPartialWindow()` then `refresh()`. Explicitly "not all panels support partial update." Differential/fast updates risk **ghosting** → periodic full refresh required.
- **BUSY pin mandatory**: every command/frame waits for BUSY to deassert before the next op (library handles it if BUSY wired correctly).
- Power: `hibernate()` / `powerOff()` for battery use.

## Panel class list (from `GxEPD2_display_selection_new_style.h`)
Select a panel by uncommenting exactly ONE class typedef matching your GoodDisplay/Waveshare part number:
- **B/W**: `GxEPD2_213_B72` (GDEH0213B72, 2.13", 122×250); `GxEPD2_290_T94` (GDEM029T94, 2.9", 128×296); `GxEPD2_420` (GDEW042T2, 4.2", 400×300); `GxEPD2_583_T8` (GDEW0583T8, 5.83", 648×480); `GxEPD2_750_T7` (GDEW075T7, 7.5", **800×480**).
- **Tri-color (B/W/R)**: `GxEPD2_154c` (GDEW0154Z04); `GxEPD2_420c` (GDEW042Z15, 400×300); `GxEPD2_750c_Z90` (GDEH075Z90, 7.5", **880×528**).
- **7-color (ACeP/Spectra)**: `GxEPD2_565c_GDEP0565D90` (5.65", 600×448); `GxEPD2_730c_GDEY073D46` (7.3", 800×480).
- Supports 60+ panels, 1.02"–13.3".

## Waveshare ESP32 driver-board pin map (from `GxEPD2_wiring_examples.h`)
**BUSY=GPIO25, RST=GPIO26, DC=GPIO27, CS=GPIO15, SCK/CLK=GPIO13, DIN/MOSI=GPIO14.** These are **non-default VSPI pins** → the sketch must remap the SPI bus. (#1 wiring gotcha.)

## Hardware cautions
- **3.3V logic only — never drive data lines from 5V** (use level shifters/dividers).
- Waveshare boards with "clever reset circuits" may need `init(115200, true, 2, false)`.
- **7.5" panels can be unreliable when powered from a 3.3V Arduino pin** — give them a solid supply.
- 7-color (5.65"/600×448) refresh ~12s and paging "gets very slow."
