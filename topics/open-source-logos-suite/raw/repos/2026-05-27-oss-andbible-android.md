# And Bible (Android, GPL)

## Key findings
- Offline-first Bible study app for Android, built on SWORD modules. Kotlin codebase (78%) with embedded JS/Vue for rich content rendering.
- Extremely active: **5.0.934 released 2026-03-11**, 620+ total releases, 13,149 commits on stable branch. One of the healthiest OSS Bible projects by any metric.
- License: **GPL-3.0** — same family as SWORD; clean compatibility.
- UI translated via Transifex into many languages.
- Features (per docs and reputation): multiple Bibles, commentaries, dictionaries, cross-references, split-screen, offline downloads, document workspaces, study pads, daily readings.

## Source notes
- **Maintainer**: Community-led on GitHub under AndBible org; small core team plus regular contributors.
- **Last active**: Very active 2026.
- **Platforms**: Android only. iOS users fall back to Pocket Sword (a separate, less active SWORD frontend).
- **License**: GPL-3.0.
- **What it does well**:
  - Most polished mobile UX in the OSS Bible space.
  - Workspace/split-screen features rival commercial mobile Bible apps.
  - SWORD-compatible — installs the same modules everyone else uses.
  - Active maintenance with rapid release cadence.
- **Gaps vs. Logos mobile**:
  - Android only — no iOS parity story in OSS-land.
  - No cloud sync of notes/highlights across devices (last I checked it was local + manual export; may have added cloud since).
  - No sermon-builder or document-authoring features.
  - Module catalog UX inherits SWORD's discoverability problem.
  - No AI/LLM-assisted study features.
- **Strategic read**: Any OSS Logos competitor that doesn't ship a great mobile app will lose. And Bible proves it's possible at high quality on Android. The iOS gap is real and is one of the clearest opportunities — Pocket Sword is sleepy by comparison.
