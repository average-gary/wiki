# WhisperX

## Architecture pipeline

1. VAD preprocessing
2. faster-whisper batched inference
3. **wav2vec2 forced alignment** (this is why WhisperX exists — Whisper's built-in word timestamps are inaccurate)
4. pyannote diarization
5. `assign_word_speakers()` joins speaker IDs to words

> Claims "<8GB GPU memory for large-v2 with beam_size=5" and "up to 70x realtime"

## Canonical 4-stage Python API

```python
import whisperx
from whisperx.diarize import DiarizationPipeline

device = "cuda"; batch_size = 16; compute_type = "float16"
model = whisperx.load_model("large-v2", device, compute_type=compute_type)
audio = whisperx.load_audio("audio.mp3")
result = model.transcribe(audio, batch_size=batch_size)

# import gc; gc.collect(); torch.cuda.empty_cache(); del model

model_a, metadata = whisperx.load_align_model(language_code=result["language"], device=device)
result = whisperx.align(result["segments"], model_a, metadata, audio, device)

diarize_model = DiarizationPipeline(token=YOUR_HF_TOKEN, device=device)
diarize_segments = diarize_model(audio)
result = whisperx.assign_word_speakers(diarize_segments, result)
```

## CLI

```bash
whisperx audio.wav --model large-v2 --diarize --hf_token YOUR_TOKEN
whisperx audio.wav --compute_type int8 --batch_size 4   # low-VRAM (Pascal!)
```

## Low-VRAM levers (relevant for 6GB Pascal)

- `--batch_size 4` (down from 16)
- `--compute_type int8` (NOT float16 — Pascal silently falls back)
- Smaller model (`--model base|medium` instead of `large-v3`)
- Sequence model loads with `gc.collect(); torch.cuda.empty_cache(); del model` between transcribe/align/diarize stages

## Output formats

SRT documented; word highlighting via `--highlight_words True`. JSON/VTT/TSV via the writer module but less prominently.
