# iroh 1.0.0-rc.1 — the API surface that 1.0 will ship

Released 2026-05-27. Last RC before stable 1.0.

## Major changes vs 1.0.0-rc.0

- **Configurable path selection** (breaking) — apps can now influence which path a connection prefers
- **Holepunching for hard-NAT servers** — previously a server behind a symmetric/hard NAT could not be reached without a relay; rc.1 makes this work
- `FourTuple` type — represents a (local-IP, local-port, remote-IP, remote-port) for selected/send paths
- **Auth tokens for relays** — relays can now require a token to accept connections
- **`AccessControl` trait replaces enum** — extensibility for custom auth logic
- **Bucket rate-limit primitive** exposed at API level
- Tracks `noq@1.0.0-rc.1` (n0's QUIC fork, in lockstep with iroh)

## What this enables for an Iroh AI server

1. **Hard-NAT mode**: most homelab boxes are behind a CGNAT or symmetric NAT — rc.1 closes that gap, more direct (relay-free) connections
2. **Relay auth tokens**: home server can run its own relay and gate by token (vs running open relay or paying for n0's relay)
3. **`AccessControl` trait**: arbitrary allowlist policies beyond simple NodeID set membership (rate limits, time windows, ALPN-conditioned policies)
4. **Bucket rate-limit**: built-in primitive for per-peer abuse mitigation
