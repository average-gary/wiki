# RFC 6238 — TOTP

The algorithmic citation for the Wesh-style time-bucketed seed-rotation rendezvous.

## Construction

```
T = floor((now() - T0) / X)
TOTP = truncate(HMAC-SHA1(K, T_bytes))
```

- `T0` = Unix epoch 0 (default)
- `X` = step size (default 30s)
- `K` = shared secret
- Truncation per RFC 4226 (HOTP)

## Stateless rendezvous

Prover and verifier independently compute the same bucket from synchronized clocks. **No shared state, no replay store needed within a window.**

## Drift tolerance

> "At most one time step is allowed as the network delay" — RFC 6238

Implementers commonly allow ±1 to ±2 buckets to handle:
- Network latency
- Clock skew between client and server
- Token-issued-at-end-of-bucket race

## Direct algorithmic parallel for the iroh app token

The Wesh-style seed-rotation rendezvous shares the same shape:

```
// TOTP:
T = floor((now() - T0) / X)
otp = HMAC-SHA1(K, T)

// Iroh app token rendezvous tag:
bucket = floor((now() - T0) / week)
tag = blake3::keyed_hash(seed, &[endpoint_id, bucket].concat())
```

The substitution is `HMAC-SHA1 → BLAKE3 keyed_hash` and `seconds-bucket → week-bucket`. Same algorithmic primitive.

## Sizing the bucket

For TOTP: 30s makes sense — humans must type the OTP; rotation should be fast enough to limit replay window but slow enough to be typeable.

For iroh app token rendezvous: bucket = **week** is the recommendation:

- Slow enough that paired devices don't churn
- Fast enough that a leaked QR has a bounded validity window
- Coarse enough that clock skew is irrelevant (no need for ±1 tolerance — week alignment is trivial)

If finer-grained revocation is needed, drop to **day** or **hour**. Below an hour, TOTP-style ±1 tolerance becomes necessary.

## Drift-tolerance recipe (lifted from RFC 6238)

```rust
fn validate(seed: &[u8; 32], endpoint_id: &EndpointId, claimed_tag: &Hash, now: u64) -> bool {
    let week = 7 * 24 * 3600;
    let current_bucket = now / week;
    for &bucket in &[current_bucket, current_bucket - 1] {  // accept current + previous
        let expected = blake3::keyed_hash(seed, &[
            endpoint_id.as_bytes(),
            &bucket.to_le_bytes(),
        ].concat());
        if &expected == claimed_tag {
            return true;
        }
    }
    false
}
```
