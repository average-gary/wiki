# Fantasy Statblocks plugin

## Snapshot
- Latest: **v4.10.3 (2026-01-21)**, 296 releases, 941 commits.
- 461 stars / 98 forks / 44 open issues.
- Stack: TypeScript 75%, Svelte 9.9%, CSS 13.5%.

## Supported systems
- **D&D 5e** (primary)
- **13th Age**
- **Pathfinder 2e** (yes, supported)
- **Daggerheart** (Critical Role's system)

## Capabilities
- Create / manage / view statblocks as code-blocks in markdown.
- Bundled `bestiary.json` ships SRD content.
- Renders stat blocks inline in notes — usable for prep AND at-table.
- Pairs with **Initiative Tracker** (same author "javalent") which can pull these into encounters with HP/AC/initiative ordering and XP calculation.

## Significance for the PF2e tool research
- Confirms a real, maintained PF2e workflow already exists inside Obsidian — the bar to clear.
- "javalent" plugin family (Fantasy Statblocks + Initiative Tracker + Leaflet + Dice Roller) is the de facto Obsidian-TTRPG stack.
- PF2e support is real but feels grafted on (D&D 5e is the design center). Statblock formatting fidelity to PF2e's three-action symbology, traits, and reactions is a known weak point.

## Limits / gaps
- No native PF2e bestiary import from Pathbuilder / Archives of Nethys.
- No automatic level/encounter-budget math the way pf2e Foundry system does.
- No conversation-style LLM-driven prep.

## Relevance
Sets the "free + open-source + offline" floor: any commercial PF2e tool needs to either (a) embed-and-extend this experience or (b) deliver enough additional value (LLM prep, native PF2e math, polished UX) that an Obsidian holdout would switch.
