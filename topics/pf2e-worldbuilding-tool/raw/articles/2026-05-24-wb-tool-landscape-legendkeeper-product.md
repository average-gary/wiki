# LegendKeeper Features

## Four pillars
1. **Lore Wiki** — auto-linking pages, GM-only secret sections, templates, offline mode, multi-format export.
2. **Interactive Maps** — nested + zoomable; pins linked to wiki entries (region → city → tavern drilldown).
3. **Timelines** — eras, moon phases, lineage tracing, parallel storylines.
4. **Whiteboards** — infinite canvas for relationship maps, magic systems, plot.

## Personas explicitly served
- Game Masters running TTRPGs
- Fiction writers
- Independent worldbuilders
- Game design teams

## AI
**None.** No mention of AI on landing or pricing pages.

## Mobile / offline
- Desktop-class web app with offline capability; mobile-web is being optimized; native apps "planned later."

## Where it wins
- Cleanest UI in the hosted-wiki category (frequent praise vs. WA's clutter).
- Nested-map UX is the category-leading feature.
- Single $9/mo Pro tier is easy to reason about.

## Where it loses
- No system-aware features (no PF2e statblocks, no encounter integration).
- No AI / generative assistance whatsoever.
- No native mobile.
- Smaller community / template marketplace than WA or Kanka.
- Hosted-only; not fully offline-first like Obsidian.

## Relevance
The clearest "what good UX looks like" reference for a PF2e tool. A PF2e-native app should match LK's cleanliness while adding the system-awareness LK explicitly avoids.
