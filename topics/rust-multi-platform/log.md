Headline findings: (a) the 4 prior-art systems converge on a two-tier secret pattern (fleet provisioning key → per-device key) and a (hardware-derived identity attribute + rotatable keypair) split; Mender's `mender-device-identity` script is the cleanest formalization. (b) **TOML cannot be the signed format** — no canonical form (ALF #397). Sign JCS-canonical JSON, deterministic CBOR (RFC 8949 §4.2), or protobuf wire format. TOML is fine as human-edited source if you parse-then-canonicalize. (c) Three-layer URI versioning (in-toto pattern: envelope-protocol prefix + statement schema URI + predicate URI) is the right shape for long-lived envelopes. (d) Two rotation idioms — dual-sign chain (TUF, DSSE multi-sig) for offline/intermittent edge; short-lived cert under long-lived root (Sigstore) for cloud-side. Cosign v3 + sigstore-c (Apr 2026) close the embedded-verification gap. (e) Pin Ed25519-IETF (RFC 8032) explicitly — three security-distinct variants exist (Brendel et al. 2021); pin one verifier impl across server and edge to dodge library-divergence bugs (Cendyne 2022; Tor + ZCash incidents). (f) Append-only is a property of the data structure, not the system — without witnesses or sufficient gossip a malicious server can fork views (split-view attack). For sub-100-device fleets, mirror to Sigstore Rekor for free witnesses-as-a-service. (g) Tile-backed transparency logs (Tessera/Rekor v2 GA Oct 2025) replace Trillian in 2026 builds; MMD = 60 sec achievable. (h) Tailscale Workload Identity Federation went GA Feb 19 2026 — eliminates static OAuth secrets in CI; SpecterOps documents the OAuth-secret exfil playbook against the older pattern. (i) Mender SD-card-clone failure mode is real: identity must be sealed to non-cloneable hardware (TPM/fuse/SoC unique ID), not "a file in /var/lib". (j) Audit-log throughput is not a bottleneck — Rekor ~2.2 entries/sec mean; 1000-device fleet × 1 event/min = ~17 events/sec, ~12% of Cloudflare Nimbus's CT write rate.
