# cross-rs/cross

## Status

- 8.2k stars, MSRV bumped to Rust 1.85.0 (2025 indicator)
- Latest tagged release v0.2.5 (Feb 2023) but actively maintained
- Container engine: Docker v20.10+ or Podman v3.4.0+ (defaults to Docker)

## Use case

> "zero setup cross compilation and cross testing... encapsulates cross toolchains and compiled libraries to produce portable binaries without modifying the host system"

## Supported targets (60+)

aarch64-unknown-linux-gnu, armv7-unknown-linux-gnueabihf, x86_64-unknown-linux-gnu, mips64, powerpc64le, riscv64gc, i686-pc-windows-gnu, thumbv7m-none-eabi, wasm32-unknown-emscripten, plus many more.

## Critical limitations

- **No macOS/Apple Darwin pre-built images** — "Additional Dockerfiles for other targets can be found in cross-toolchains. These include MSVC and Apple Darwin targets, which we cannot ship pre-built images of."
- **No MSVC Windows images** (licensing)
- **QEMU testing is sequential and slow** — multi-threaded tests can hang

## Usage

```bash
cargo install cross --git https://github.com/cross-rs/cross
cross build --target aarch64-unknown-linux-gnu
cross test --target mips64-unknown-linux-gnuabi64
cross rustc --target powerpc-unknown-linux-gnu --release -- -C lto
```

## Cross.toml configuration

```toml
[workspace.metadata.cross.target.aarch64-unknown-linux-gnu]
pre-build = [
    "dpkg --add-architecture $CROSS_DEB_ARCH",
    "apt-get update && apt-get --assume-yes install libssl-dev:$CROSS_DEB_ARCH"
]

[workspace.metadata.cross.target.armv7-unknown-linux-gnueabi]
image = "my/image:latest"
```

## Cross-in-container

```bash
docker run -v /var/run/docker.sock:/var/run/docker.sock -v .:/project \
  -w /project my/development-image:tag \
  cross build --target mips64-unknown-linux-gnuabi64
```

with `CROSS_CONTAINER_IN_CONTAINER=true`.

## Decision matrix

- Linux-on-Linux cross → **cross** (or native cargo)
- Apple Darwin cross-from-Linux → **cargo-zigbuild** (cross can't do it)
- Windows MSVC cross-from-Linux → **cargo-xwin**
- Windows MinGW cross → **cross**
