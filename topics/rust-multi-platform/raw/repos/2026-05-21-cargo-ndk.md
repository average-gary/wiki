# cargo-ndk

## What it does

> "a cargo extension that handles all the environment configuration needed for successfully building libraries or binaries for Android from a Rust codebase"

Replaces hand-rolled `.cargo/config.toml` linker stanzas, `cargo-mobile`, and Mozilla's `rust-android-gradle` Gradle plugin (which still works but heavier).

## Setup

```bash
rustup target add \
    aarch64-linux-android \
    armv7-linux-androideabi \
    x86_64-linux-android \
    i686-linux-android

cargo install cargo-ndk
```

## Build (4 ABIs into jniLibs)

```bash
cargo ndk -t arm64-v8a -t armeabi-v7a -t x86_64 -t x86 \
  -o ./android-lib/src/main/jniLibs build --release
```

Output structure: `./android-lib/src/main/jniLibs/<abi>/lib<crate>.so` — exactly what an Android Library Gradle module wants.

## Subcommands

- `cargo ndk` (build)
- `cargo ndk-test` (tests on emulator/device)
- `cargo ndk-env` (dump env vars for IDE integration)

## Auto-exports

`ANDROID_ABI`, `ANDROID_PLATFORM`, sysroot paths, correct linker per target (`aarch64-linux-android21-clang`, etc.).

## NDK version

- Auto-detects most recent NDK if installed via Android Studio in default location
- Override via `ANDROID_NDK_HOME`
- 2025/2026: NDK r27 is the LTS line; r25/r26 work but Rust may drop them per [Android Tier 2 LTS policy](rust-android-platform-support)

## Pipeline position

`rustc → cargo-ndk → .so files in jniLibs → Gradle assembleRelease → .aar`

Then UniFFI generates Kotlin bindings from your `.udl` or proc-macros. AAR + Kotlin bindings = consumable library for the app module.
