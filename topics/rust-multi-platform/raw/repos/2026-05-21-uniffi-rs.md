# mozilla/uniffi-rs

## Languages

| Tier | Languages |
|------|-----------|
| First-party | Kotlin, Swift, Python (Ruby = legacy partial) |
| Third-party | JS, Go, C#, Dart, Java, Node.js |

## Two interface styles

Both first-class, documented side-by-side per feature:
- **UDL files** — WebIDL-based `.udl` definition file
- **proc-macros** — `#[uniffi::export]`, `#[derive(uniffi::Record)]`, etc. (modern default for greenfield)

## Important framing (from docs)

> "this tool will not help you ship a Rust library to these platforms, but it will help you avoid writing bindings code by hand"

— packaging (xcframework / AAR) is YOUR problem. UniFFI just emits the language-side code.

## Async support

- Rust `async fn` → Swift `async`/`await`, Kotlin `suspend fun`, Python `asyncio`
- Foreign side supplies the executor (no Rust runtime forced)
- **No built-in cancellation** — DIY via flag + `cancel()` method
- Async trait methods need `async-trait`

## Production validation

- "extensively used by Mozilla in Firefox mobile and desktop browsers"
- Independent shippers: Bitwarden, Matrix Rust SDK / Element X, plus others
