# Kubernetes ServiceAccount Token Evolution (2025-2026)

Establishes the upstream "no long-lived secrets at the edge" baseline that downstream distros (k3s, k0s, RKE2) inherit.

## Timeline

| Release | Date | Change |
|---|---|---|
| **k8s 1.33** | 2025-04 | Workload Identity for image pulls — alpha. Projected SA tokens replace static `imagePullSecrets`. |
| **k8s 1.34** | 2025-08 | SA tokens for image pulls graduated to **Beta** — ready for fleet rollout testing |
| **k8s 1.35** | 2025-12 | SA token integration for CSI drivers via projected volumes — **Beta** |

## Mechanism

- Bound, audience-scoped tokens
- Signed by the cluster, time-limited (typically 1 hour, refreshed via projected volume)
- No long-lived registry credentials or CSI secrets on disk

## Why it matters

Defines what "fleet identity" means in 2026: bound tokens with audience claims. Other platforms are converging on this:

- Tailscale Workload Identity Federation (Feb 2026 GA)
- AWS IAM Roles Anywhere
- HashiCorp Vault dynamic secrets
- GitHub Actions OIDC + Sigstore Fulcio
