# Cosign v3 (Oct 8, 2025) + sigstore-c (Apr 1, 2026)

Two-day prelude to Rekor v2; paired migration.

## Cosign v3

- Released **Oct 8, 2025** — two days before Rekor v2 GA
- **Offline verification** emphasized — critical for edge fleets with intermittent connectivity
- **Tile-based transparency log** client implementation — required to talk to Rekor v2

## sigstore-c (Apr 1, 2026)

- **C89-compatible client library** for embedded/edge devices that can't run Go binaries
- Makes signed-envelope verification feasible on resource-constrained edge nodes
- Direct enabler for "Cosign verification on Cortex-M4-class fleet members"

## Adoption signals (per News/Trends agent)

## Why it matters

Until Cosign v3 / sigstore-c, the Sigstore stack required a fully-online OIDC + Fulcio + Rekor round-trip and a Go runtime — effectively gating it from edge fleets. The 2025-Q4 / 2026-Q2 releases close that gap. Edge fleets shipping in 2026+ should reconsider whether the "long-lived ed25519 + rotation" model is still the right default vs short-lived Sigstore identity per attestation.
