# Rekor v2 GA — Oct 10, 2025

The reference design for high-throughput append-only audit logs in 2026.

## Architecture change

Rekor v2 replaces the Trillian backend with **Tessera tile-backed transparency log** — slashes operational cost and storage. Public instance supports higher QPS and CDN-cacheable read paths.

## Breaking changes

- Removed signed timestamps with proofs (now separate)
- Removed built-in search index (separate service)
- Removed attestation storage
- Only `hashedrekord` and DSSE entry types supported (9 legacy types removed)

## New URL scheme

`logYEAR-rev.rekor.sigstore.dev` replacing the single endpoint — clients must update.

## Witness cosigning

To be integrated directly into the log for stronger append-only guarantees (forthcoming at GA time). **This addresses the split-view weakness contrarian-6 documented** — single-signer logs are insufficient.

## Migration

Public instance runs v1 + v2 in parallel; v1 deprecation requires **one-year notice**.

## Production scale (per Data agent)

Rekor sample log index sighted Oct 2025: **262,231,672** → ~262 M entries

Implied growth from launch (Mar 2021) to Oct 2025 (~46 months): ~5.7 M entries/month, ~190k/day, ~2.2 entries/sec average. **BigQuery public dataset `rekor` launched 2025-08-15** for researchers.

## Why it matters

Tile-backed logs and witness cosigning are the **new gold standard** for "audit logs / signed envelopes" — fleet operators verifying signed artifacts must update clients. If you build an append-only audit log in 2026, copy this architecture, not RFC 6962-era Trillian.
