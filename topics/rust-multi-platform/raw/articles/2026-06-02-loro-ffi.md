`loro-ffi` is the Foreign Function Interface umbrella for Loro, **built with
Mozilla's UniFFI** (the same framework christ-is-lord uses for its own
`logos_ffi` crate). Bindings exposed: Swift (`loro-swift`), Python
(`loro-py`, via pyo3), React Native (`loro-react-native` Turbo Module), plus
community-maintained C# (`loro-cs`) and Go (`loro-go`). The repo publishes
11 tags. Notably absent from the first-party list: a dedicated
`loro-kotlin` / Android Gradle distribution — there is no `loro-kotlin`
repository under `loro-dev` (the `github.com/loro-dev/loro-kotlin` URL returns
404 as of 2026-06-02). React Native covers an Android path indirectly, but a
pure-Kotlin/JVM artifact comparable to yrs's `ykt` is not yet first-party.
MIT-licensed; written 100% Rust.
