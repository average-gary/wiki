# iroh 1.0.0-rc.0 ‚Äî The first release candidate

First 1.0 release candidate after 4 years and 50+ releases. Signals API
stabilization commitment.

## API surface cleanup

- `ConnectionInfo` ‚Üí `WeakConnectionHandle`.
- Path API redesigned to `Path<'_>` / `PathEventStream`.
- MSRV bumped to **Rust 1.91**.
- "We eliminated almost all types from pre-1.0 crates in reexports and exported
  types, with only a very small number left."

## Custom transports gated

- Moved behind `unstable-custom-transports` feature flag ‚Äî they will remain
  unstable post-1.0. (Tor / Nym / BLE custom transports ‚Äî see 0.97 release
  notes ‚Äî are on this surface.)

## Discovery extracted

- `DhtAddressLookup` ‚Üí separate `iroh-mainline-address-lookup` crate.
- `MdnsAddressLookup` ‚Üí separate `iroh-mdns-address-lookup` crate.

## Production claim

> "Iroh is open source, and already running in production on hundreds of
> thousands of devices."

## Implications for SV2

- **Target the rc / 1.0 API** in `feat/iroh-transport`, not 0.96 or earlier.
  The rename churn (Node‚ÜíEndpoint, NodeAddr‚ÜíEndpointAddr, Discovery‚ÜíAddressLookup,
  ConnectionType‚ÜíTransportAddr) is mostly behind us.
- MSRV 1.91 is fine ‚Äî sv2-apps already tracks recent stable.
- Don't depend on `unstable-custom-transports` from a stable SV2 transport ‚Äî
  reach for it only via an explicit feature flag that pool operators opt into
  (e.g., `iroh-tor` feature).
- DHT and mDNS discovery require explicit dependencies as of 1.0 ‚Äî `presets::N0`
  no longer pulls them in transparently.
