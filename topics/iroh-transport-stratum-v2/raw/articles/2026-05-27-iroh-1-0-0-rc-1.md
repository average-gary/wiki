# iroh 1.0.0-rc.1 — The last one

Final release candidate before 1.0 stable. Two weeks after rc.0. Folds in
community feedback from rc.0 and finalizes the API surface n0 expects to
ship as 1.0.

## Major features

- **Hard-NAT holepunching**: server-side endpoint-dependent NAT mapping is
  now traversable.
- **Configurable path selection** (`PathSelector` trait, gated behind
  `unstable-custom-transports`): apps can override how iroh picks among
  candidate paths. Includes new types `PathSelection`,
  `PathSelectionContext`, `PathSelectionData`, `FourTuple`. Not 1.0-stable.
- **Pluggable relay access control**: `AccessControl` trait (with
  `on_connect` / `on_disconnect` hooks and a per-connection `ConnectionId`)
  replaces the rc.0 `AccessConfig` enum. Each relay connection can be
  proactively disconnected.
- **Faster endpoint closure**: shutdown skips the QUIC drain when feasible —
  near-instant or ~1 RTT in the typical case.

## Breaking changes

### `iroh`

- Removed `IncomingLocalAddr`; replaced by `LocalTransportAddr` (the `Relay`
  variant is now tuple-based).
- `PathEvent` variants now `#[non_exhaustive]`.
- `CustomSender::poll_send` gains a `src: Option<&CustomAddr>` parameter.
- New `Builder::path_selector` method.
- Under `unstable-custom-transports`: `PathSelector` / `PathSelection` /
  `PathSelectionContext` / `PathSelectionData` / `FourTuple` added.

### `iroh-relay`

- Removed `AccessConfig` enum and `AccessCheck` type alias.
- `Client` is no longer public.
- `RelayConfig::access` is now `Arc<dyn DynAccessControl>`.
- `Access::Deny` is now a struct variant with optional `reason` field;
  `Access` is no longer `Copy`.
- `ClientRequest::new` gains a protocol-version parameter.
- `Config::new` replaces `EndpointId` with `OnDisconnectGuard`.
- New: `AccessControl`, `DynAccessControl`, `AllowAll`, `ConnectionId`,
  `OnDisconnectGuard`. `Bucket` rate-limit primitive is now public.

### `iroh-base`

- `SignatureParsingError` is now re-exported.

## Dependency bumps

- `noq@1.0.0-rc.1` (n0's Quinn fork follows iroh's rc cadence).
- `netwatch@0.18.0` (transitive — bumped from 0.17 in our lockfiles).

## Stability fixes

- Remote-state actor races corrected.
- Path-abandonment scope isolation tightened.
- Static musl build CI fixes.

## For SV2: pure version bump; no API churn in the surfaces SV2 uses

Verified by `cargo build --features iroh-transport` on all four workspaces
(`stratum-apps`, `pool-apps`, `miner-apps`, `integration-tests`) plus the
full integration test suite (`pool_integration_iroh`, `jd_integration_iroh`,
`translator_integration_iroh`, `fallback_iroh_to_tcp`) on rc.1. **Zero
source code changes required.**

Why each rc.0→rc.1 breaking change misses SV2:

- **Path API `FourTuple` refactor**: SV2 doesn't consume `Path<'_>` events
  or implement `CustomSender`. The new `path_selector` builder method is
  optional; `presets::N0` still picks a path automatically.
- **`iroh-relay` `AccessControl` trait**: SV2 runs as iroh **clients**, not
  as relay operators. We don't import `iroh-relay` directly.
- **`IncomingLocalAddr` → `LocalTransportAddr` rename**: SV2 doesn't
  inspect incoming connections' local-addr discriminants; we use
  `Connection::remote_id()` for peer identity.
- **`#[non_exhaustive]` on `PathEvent`**: SV2 doesn't match on `PathEvent`
  variants.
- **`SignatureParsingError` re-export**: SV2 doesn't structurally match on
  iroh-base errors.

The surfaces SV2 *does* consume — `Endpoint::builder(presets::N0)`,
`.alpns(...)`, `.secret_key(...)`, `.bind()`, `EndpointId`, `EndpointAddr`,
`RelayMode`, `RelayUrl`, `SecretKey`, `Connection::accept_bi()` /
`open_bi()`, `presets::N0` — are unchanged in rc.1.
