# p2poolv2

## Architecture

- **Downstream**: SV1 + SV2 listener for miners.
- **Upstream**: bitcoind RPC directly (no pool).
- **Payout**: share-chain with uncle accounting + atomic-swap settlement.

## Why it's not a reverse-translator customer

p2poolv2's mission is to *replace* centralized pools. There is no upstream pool to translate to. A reverse translator only matters if you want to mine to an SV1 *pool*; p2pool is not a pool.

## Adjacent relevance

- The SV1+SV2 dual-listener pattern is the inverse of what a reverse translator does (it terminates downstream protocols rather than translating upstream protocols), but the listener scaffolding is reusable as code reference.
- Eliminates one hypothesized customer segment for the reverse translator (decentralized share-chain operators).
